# Amber v0.3.2 账户面板与访问队列开发设计

> 状态：实现完成，待发布
> 目标版本：v0.3.2
> 数据库版本：schema 10
> 基线版本：v0.3.1 (`fd6df59`)
> 更新日期：2026-07-17

## 1. 目标

v0.3.2 聚焦账户管理和本地网关调度体验，完成四项核心修改：

1. 将 Base URL + API Key、ChatGPT 授权登录和 JSON 导入合并为一个“导入账号”入口。
2. 将双列大卡片改为单行账户卡片，详情与完整用量放入独立详情弹窗。
3. 增加账户调用开关；关闭后调度器不得再选择该账户，严重鉴权异常可自动关闭。
4. 增加每账户最大并发与有界等待队列；并发占满后排队，队列满、等待超时或客户端取消时明确退出。

本设计参考 `D:\Study\other\sub2api\sub2api` 中账户 `concurrency`、`schedulable`、并发槽位和等待计数的职责划分，但使用 Amber 现有单机 Go + SQLite 架构实现，不引入 Redis、分布式锁或后台管理系统。

## 2. 版本边界

### 2.1 v0.3.2 必须完成

- 单一“导入账号”按钮和三种导入方式选择弹窗。
- JSON 文件支持单文件、多文件、JSON 数组、`{"accounts": [...]}` 和连续 JSON/JSONL。
- 所有 JSON 导入继续经过预览、SHA-256 内容绑定和单事务提交。
- 账户列表一行一个账户，窄窗口可降级为两行信息布局但仍保持一账户一项。
- 详情弹窗展示身份、状态、用量、时间、代理、并发配置和账户操作。
- 明确的账户调用开关，关闭后立即从新请求调度中排除。
- 重复鉴权失败和明确账户失活错误的自动停用策略。
- 每账户最大并发、等待队列上限、当前并发和当前等待人数。
- 队列满返回结构化 429；等待超时返回结构化 503；客户端取消立即清理等待计数。
- schema 10 自动迁移、迁移前备份和云端加密账户负载兼容。
- 中英文 i18n、Go 单元/竞态测试、Vue 单测和 Playwright E2E。

### 2.2 v0.3.2 明确不做

- 不实现多进程或跨设备分布式队列。
- 不引入 Redis、消息队列或外部调度服务。
- 不保证流式请求断线后的上游计费回滚。
- 不把 M3 幂等写入与超时安全提前到 v0.3.2；M3 仍属于 v0.3.3。
- 不恢复 OAuth Worker 直连共享；设备回源共享仍属于 v0.4.0。
- 不在本版本提供队列优先级、VIP 插队或按模型拆分并发池。

## 3. 统一导入入口

### 3.1 页面入口

账户页头部只保留一个主要命令：`导入账号`。

点击后打开方式选择弹窗，提供三个可点击选项：

- `Base URL + API Key`：进入现有 API Key 表单。
- `ChatGPT 授权登录`：进入现有浏览器 OAuth 登录流程。
- `JSON 文件`：进入现有安全导入预览流程。

空状态使用同一个“导入账号”入口，不再单独显示多个按钮。

### 3.2 JSON 文件能力

文件选择器增加 `multiple`，允许同时选择多个 `.json`、`.jsonl` 或文本文件。

前端处理规则：

1. 每个文件单独执行 UTF-8/UTF-16 BOM 解码。
2. 按用户选择顺序用换行连接为一个连续 JSON 数据流。
3. 总体内容仍受后端 10 MiB 上限保护。
4. 预览和提交使用完全相同的合并内容，继续校验 `X-Import-Preview-SHA256`。
5. 文件列表只展示文件名、数量和总大小，不在日志中输出凭据内容。

后端现有解析器已经支持：

- 单个账户对象。
- JSON 数组中的多个账户。
- `{"accounts": [...]}` 包装格式。
- 连续 JSON 文档和 JSONL。
- sub2api 备份、Codex CLI `auth.json`、OAuth 与 API Key 混合数据。

因此本轮不新增不安全的前端凭据解析，只扩展多文件聚合和对应测试。

## 4. 账户行卡片

### 4.1 列表可见信息

每个账户行只展示扫描和操作所需的关键字段：

- 头像首字母、邮箱或 API Key 名称。
- 账户类型与套餐/Base URL 摘要。
- 当前状态 badge；自动停用时显示原因提示。
- 最近成功时间。
- 总 Token 与估算费用。
- 当前并发/最大并发、当前等待/队列上限。
- 调用开关。
- `查看详情` 按钮。

不得在列表直接铺开完整用量窗口、代理选择器和全部操作按钮。

### 4.2 详情弹窗

详情弹窗包含以下不嵌套卡片的分区：

- 身份：邮箱、账户 ID、类型、套餐、Base URL、创建时间、令牌到期。
- 健康：状态、原因、连续失败、最近使用、最近成功、下次重试。
- 累计用量：请求数、输入/缓存/输出/推理 Token、总 Token、估算费用。
- 上游窗口：Codex 主窗口与次窗口使用率、窗口长度和重置倒计时。
- 调度：最大并发、等待队列上限、当前并发、当前等待人数。
- 网络：绑定代理。
- 操作：测试连接、刷新/重新登录、共享、强制恢复、删除。

删除仍必须经过二次确认；凭据值永不展示。

## 5. 账户启停与自动停用

### 5.1 手动开关

v0.3.2 复用现有 `AccountDisabled` 状态表达调用开关关闭：

- 关闭：`status=disabled`，`status_reason=manually_disabled`。
- 打开：恢复为 `active`，清除限流时间、失败计数和下次重试。
- 调度器在每次候选选择时排除 `disabled`。
- 已经开始的请求不被强制中断；开关只影响后续新请求。

临时 `rate_limited`、`refresh_failed` 和 `pending_validation` 仍属于开启状态，但可能因恢复时间或凭据条件暂时不可调度。

### 5.2 自动停用规则

自动停用仅用于凭据或账户本身的严重异常：

- 明确错误包含账户所有者失活/账户停用语义时立即停用。
- 连续 3 次确定的 401/403 或 OAuth 刷新鉴权失败后停用。
- 成功的端到端请求把连续鉴权失败计数清零。
- 429 只进入限额窗口，不自动停用。
- 网络、DNS、TLS、代理、5xx 和客户端取消不增加鉴权失败计数，不自动停用。

自动停用使用可本地化的稳定原因码，原始上游响应仅保留在受限诊断日志中；账户列表不直接显示整段 HTML/JSON 错误。

### 5.3 恢复

用户可通过开关或“强制恢复”重新启用。OAuth 凭据无效时应引导重新登录；API Key 账户需由用户修正凭据后再恢复。

## 6. 每账户并发与等待队列

### 6.1 持久化字段

schema 10 为 `accounts` 增加：

- `max_concurrency INTEGER NOT NULL DEFAULT 3`
- `queue_capacity INTEGER NOT NULL DEFAULT 20`

合法范围：

- 最大并发：1 到 100。
- 等待队列上限：0 到 1000；0 表示并发满时不等待。

字段加入账户云同步密文。旧设备解密新负载时忽略未知字段；新设备读取旧负载时使用默认值 3/20。

### 6.2 运行时状态

调度器在内存维护：

- `in_flight[account_id]`
- `waiting[account_id]`
- 容量变化通知通道

控制接口返回运行时字段：

- `in_flight`
- `waiting`

运行时计数不写入 SQLite，也不进入云同步。

### 6.3 请求流程

1. 网关读取所有可调度账户。
2. 按当前策略排序，并选择第一个仍有并发槽位的账户。
3. 找到槽位后原子增加 `in_flight`，请求结束时通过幂等 release 释放。
4. 所有账户均满时，按排序结果寻找仍有等待容量的账户并登记等待。
5. 任一槽位释放后唤醒等待请求，重新按最新状态选择可用账户。
6. 获取槽位后减少原登记账户的等待计数。
7. 客户端取消、服务器关闭或等待超时必须减少等待计数。

等待最长 60 秒。该时长本版本保持服务端常量，后续可根据实际使用数据决定是否开放全局设置。

### 6.4 失败响应

- 所有候选队列都已满：HTTP 429，错误码 `account_queue_full`，建议客户端稍后重试。
- 60 秒内没有槽位：HTTP 503，错误码 `account_queue_timeout`。
- 客户端取消：停止等待，不继续访问上游。
- 没有启用或凭据可用的账户：保持 HTTP 503，错误码 `no_account`。

失败响应不得占用账户调用次数，也不得写入上游用量。

### 6.5 并发与故障转移

- 首次账户选择必须经过等待队列。
- 首次上游失败后尝试下一账户时只进行非阻塞槽位获取；已满账户直接跳过，避免一个失败请求重复占用多个等待队列。
- release 必须幂等，流式响应正常结束、错误、panic 防护和客户端取消路径都必须释放。
- 账户关闭不强杀已持有槽位的请求；请求完成后正常释放。

## 7. 控制接口

### 7.1 账户列表

`GET /control/accounts` 的账户对象新增：

- `max_concurrency`
- `queue_capacity`
- `in_flight`
- `waiting`

### 7.2 调用开关

继续使用：

`POST /control/accounts/{id}/status`

请求体只允许 `active` 或 `disabled` 由界面开关调用；内部测试与恢复流程可继续使用其他现有健康状态。

### 7.3 调度限制

新增：

`PUT /control/accounts/{id}/limits`

请求体：

```json
{
  "max_concurrency": 3,
  "queue_capacity": 20
}
```

响应返回更新后的脱敏账户对象。超出范围返回 HTTP 400 和稳定错误码 `invalid_account_limits`。

## 8. 数据迁移与云同步

### 8.1 schema 10

迁移必须：

- 使用 `addColumnIfMissing` 保持重复执行安全。
- 为旧账户补默认值 3/20。
- 更新 `accounts_cloud_dirty` 触发器，把两个字段加入脏标记列。
- 迁移失败时恢复升级前数据库备份。

### 8.2 云同步

`accountPayload` 增加两个可选整数字段。验证规则为 1..100 和 0..1000；字段缺失或为零的旧负载分别回退到 3/20。

`ApplyCloudAccount` 创建和更新路径必须保留这两个值，并避免云同步覆盖运行时计数。

## 9. 前端交互与可访问性

- 方式选择弹窗中的三项必须支持键盘聚焦和 Enter/Space 激活。
- 调用开关使用 checkbox/switch 语义并有明确可访问名称。
- 详情弹窗支持 Esc 关闭，操作进行中不得误关闭。
- 行卡片在窄窗口不产生横向滚动，长邮箱/Base URL 使用省略号并提供 title。
- 操作按钮使用现有图标库；陌生图标提供 title。
- 中英文文案同步增加，不在模板中硬编码业务文案。

## 10. 测试计划

### 10.1 Store 与迁移

- schema 9 升级到 10 后默认并发/队列正确。
- 新数据库直接创建 schema 10。
- 限制更新范围校验和持久化。
- 手动关闭后账户不被调度。
- 三次鉴权失败自动停用；一次成功清零。
- 明确账户失活立即停用。
- 网络、代理、429 和 5xx 不触发自动停用。
- 云同步来回保留并发与队列设置。

### 10.2 Scheduler 与 Gateway

- 并发槽位不超过每账户上限。
- 槽位释放后等待请求继续执行。
- 队列上限生效并返回 429。
- 等待超时和客户端取消清理计数。
- 多账户时选择仍有容量的账户。
- quota-aware、round-robin 和 failover 顺序保持兼容。
- 故障转移不会突破下一账户并发限制。
- `go test -race` 下计数无竞争且 release 幂等。

### 10.3 导入

- 单文件单账户。
- 单文件账户数组。
- 单文件 `accounts` 包装数组。
- 多文件混合对象、数组和 JSONL。
- UTF-8/UTF-16 文件组合。
- 多文件预览与提交 SHA 一致。
- 批次冲突时整批不写入。

### 10.4 Vue 与 E2E

- 页面只显示一个“导入账号”入口。
- 三种方式分别打开正确流程。
- 多文件选择显示数量并完成预览。
- 一账户一行，详情默认关闭。
- 详情弹窗展示完整用量并可保存限制。
- 开关关闭/打开调用正确接口并刷新状态。
- 自动停用原因可见。
- 桌面和窄窗口无横向溢出。
- 既有 API Key 共享和 OAuth 共享止损入口仍可从详情弹窗访问。

## 12. 完成定义

v0.3.2 只有在以下条件同时满足时完成：

1. 用户只通过一个入口即可选择三种账号导入方式。
2. 单文件、多文件和单文件多账户导入均经过同一安全事务流程。
3. 账户列表可以快速扫描，完整信息只在详情中展示。
4. 手动关闭和严重鉴权异常自动关闭都能阻止后续调度。
5. 每账户并发永不超过配置值，队列计数在成功、失败、取消和超时后均归还。
6. 既有调度策略、代理绑定、测试、刷新、共享和云同步无回归。
7. 全部质量门禁通过并生成可校验的 v0.3.2 安装包。
