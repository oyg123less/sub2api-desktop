# Amber v0.3.1 可靠性修复计划

> 状态：实现完成，待发布验证
> 目标版本：v0.3.1
> 后续版本：v0.3.3（M3 幂等与超时安全）、v0.4.0（OAuth 设备回源共享）
> 更新日期：2026-07-17

## 1. 背景

v0.3.1 已完成云账户、端到端加密同步、共享治理、模型价格和桌面端管理功能，但实际验证发现两类发布阻塞问题：

1. Amber Cloud 连接存在间歇性超时。对生产 Worker `/health` 连续探测 6 次，4 次成功、2 次在建立连接前超时；当前客户端单次失败即终止同步。
2. OAuth 云共享会由 Cloudflare Worker 请求 ChatGPT 内部 Codex 接口。Worker 无法复现用户本地 TLS 指纹和网络出口，生产请求被上游返回 HTML 403。

本地待上传数据通过 `sync_dirty` 持久化，网络失败不会删除数据。本次观测到的“待上传 1”是账户记录，仍安全保存在本地数据库中。

## 2. 版本边界

### 2.1 v0.3.1 本轮必须完成

- 云请求安全重试和错误阶段分类。
- Windows 静态系统代理识别，并保留环境变量代理优先级。
- 后台同步失败后的短周期自动恢复。
- 同步状态增加失败阶段、错误码、尝试时间、连续失败次数和下次重试时间。
- 云账户页面实时刷新状态，提供清晰的中文错误和立即重试入口。
- 禁止创建当前架构无法可靠工作的 OAuth Worker 直连共享。
- Worker 对已有 OAuth 共享返回结构化错误，不再请求 ChatGPT，也不消耗共享配额。
- Worker 将上游 HTML 错误转换为安全 JSON，禁止把整页 HTML 透传给客户端。
- API Key 云共享保持可用。

### 2.2 v0.3.1 明确不做

- 不实现 OAuth 设备回源 WebSocket 隧道。
- 不把 TLS 指纹、网络出口或完整客户端环境上传给 Worker。
- 不新增 D1 幂等请求表、批次回执表或成功结算配额模型。
- 不改变现有 vault 密文格式、主密码派生方式和本地数据库 Schema。
- 不修改尚未完成验证的 Resend 发件域配置。

### 2.3 v0.3.3：M3 幂等与超时安全

以下内容延后到 v0.3.3：

- `PUT /v1/vault/batch` 使用 `Idempotency-Key`。
- D1 保存短期批次回执，超时重试返回原结果。
- 服务端已写入但客户端未收到响应时，避免重复版本和假冲突。
- 拉取分页游标和本地应用建立更细粒度事务检查点。
- 共享配额采用“预占、成功结算、失败释放”模型。
- 同步中途退出、响应丢失和重复提交的故障注入测试。

### 2.4 v0.4.0：OAuth 设备回源共享

- Worker 负责 Guest Key、配额和设备路由。
- 账号拥有者的 Amber Agent 主动建立加密 WebSocket。
- OAuth token 只保存在拥有者本地。
- 最终 ChatGPT 请求由拥有者设备发出，保留本地 TLS、代理和网络出口。
- 设备离线时返回 `owner_device_offline`。

## 3. M1：云连接可靠性

### 3.1 错误分类

`CloudError` 增加稳定字段：

- `code`：程序判断使用的错误码。
- `stage`：`dns`、`connect`、`tls`、`timeout`、`response`、`http`、`local`。
- `retryable`：是否允许自动重试。
- `attempt`：本次请求的尝试次数。

界面禁止直接依赖底层英文错误字符串。日志仅记录错误类型、阶段、HTTP 状态和请求 ID，不记录 token、密码、邮箱、密文和请求正文。

### 3.2 重试策略

v0.3.1 只自动重试不会产生重复写入的请求：

- GET/HEAD：最多 3 次。
- 网络错误、超时、HTTP 429 和 5xx：允许重试。
- HTTP 400、401、403 和 409：不重试。
- 间隔：500ms、1.5s，并加入小幅随机抖动。
- `Retry-After` 存在时优先遵循服务端值。

`PUT /v1/vault/batch` 不在 v0.3.1 内进行响应不明时的自动重放；其安全幂等重试属于 v0.3.3。

### 3.3 代理策略

优先级如下：

1. `HTTP_PROXY`、`HTTPS_PROXY`、`NO_PROXY` 环境变量。
2. Windows Internet Settings 中启用的静态代理。
3. 直连。

v0.3.1 不解析 PAC 脚本。PAC 和用户可选 Amber 代理在后续版本单独设计，避免补丁版本引入不可测试的代理路由行为。

## 4. M2：同步恢复状态机

- 保留 `opMu`，确保手动同步、后台同步和创建共享触发的同步不会并发执行。
- 启动后 10 秒执行首次后台同步。
- 成功后保持 5 分钟周期。
- 失败后按 5 秒、15 秒、1 分钟、5 分钟退避，成功后归零。
- 所有 Sync 返回路径统一记录成功或失败，避免本地加密、数据库和冲突错误漏记。
- `last_sync_at` 只表示成功时间。
- 新增 `last_attempt_at`、`last_error_code`、`last_error_stage`、`consecutive_failures`、`next_retry_at`。
- 网络失败不得清除 `sync_dirty`、tombstone 或冲突记录。
- 本轮重试状态保存在内存；Amber 重启后由持久化脏数据和启动同步恢复。

## 5. M4：云账户页面

- 页面每 5 秒静默刷新云状态；窗口卸载时清理计时器。
- 手动同步失败后立即重新读取状态，不再只显示 toast。
- 错误卡片显示本地化阶段、最近尝试时间、连续失败次数和下次重试倒计时。
- 增加“立即重试”按钮。
- 同步成功后自动清除旧错误卡片。
- 保留上次成功时间、待上传数量和冲突数量。
- 中英文 i18n 必须同时更新。

## 6. M5：共享功能止损

### 6.1 新建共享

- API Key 账户：继续允许创建云端共享。
- OAuth 账户：桌面端拒绝创建并提示需要 v0.4.0 设备回源共享。
- 不上传新的 OAuth access token 到 Worker。

### 6.2 已有共享

- Worker 在解密凭据后、占用配额前识别 OAuth 类型。
- 返回 JSON 错误码 `oauth_device_relay_required`。
- HTTP 状态使用 409，表示当前共享传输模式不受支持。
- 不访问 `chatgpt.com`，不增加 `used_requests`。

### 6.3 上游错误

- API Key 上游返回 HTML 时，转换为 `share_upstream_rejected` JSON。
- 不向好友客户端返回上游 HTML、Authorization 或敏感响应头。
- 仍记录模型、HTTP 状态和延迟等非敏感元数据。

## 7. 测试计划

### 7.1 Go

- GET 请求前两次网络失败、第三次成功。
- 非幂等 PUT 网络失败时不重放。
- DNS、连接、TLS 和超时错误映射。
- 后台同步退避序列和成功复位。
- 同步失败后脏数据仍存在。
- 手动同步与后台同步不并发。
- Windows 系统代理解析单元测试。

### 7.2 Cloud Worker

- OAuth 共享返回结构化 409。
- OAuth 拒绝不消耗配额、不产生上游请求。
- API Key Responses 和 Chat Completions 保持通过。
- HTML 403 转换为安全 JSON。
- Guest Key、配额、撤销和过期测试保持通过。

### 7.3 Vue 与 E2E

- 同步错误阶段中英文文案可编译。
- 错误卡片显示失败次数和重试倒计时。
- 点击立即重试会刷新状态。
- 后台状态更新后旧错误自动消失。
- OAuth 共享入口给出明确提示。

## 8. 发布门禁

- `gofmt -l` 无输出。
- `go vet ./...` 通过。
- `go test -race ./...` 通过。
- `npm run build`、`npm test` 通过。
- Playwright 全量通过。
- Cloud `npm run typecheck`、`npm test` 通过。
- Rust fmt、clippy、test 通过。
- 重新构建 sidecar 和 NSIS 安装包。
- 不运行安装包，不卸载用户已安装的 Amber。
- 核对版本、文件大小和 SHA-256。
- 推送源码与 `v0.3.1` 标签，安装包发布到 GitHub Release，不提交进 Git 历史。

## 9. 完成定义

v0.3.1 只有在以下条件全部满足时才允许发布：

1. 间歇性 GET 连接失败可以在一次同步操作内恢复。
2. 最终失败时用户能看到具体阶段和自动重试时间。
3. 待上传数据在失败、重启和恢复过程中不丢失。
4. OAuth 云共享不再产生 ChatGPT HTML 403。
5. API Key 云共享无回归。
6. 所有质量门禁通过并生成可校验安装包。
